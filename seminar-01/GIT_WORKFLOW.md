# Краткий cheat-sheet по Git

## Шаг 1: Создание новой ветки

**Используйте комбинированную команду** (создание и переключение одновременно):

```bash
git checkout -b seminar-01-solutions
```

Проверьте, что вы на новой ветке:

```bash
git branch
```

Текущая ветка будет отмечена звёздочкой (*).

---

## Шаг 2: Внесение изменений

Откройте любой из файлов задач (Task1-Task5) и решите его. Например, реализуйте Task1SumOfTwoNumbers.java.

---

## Шаг 3: Добавление файлов в staging area

Добавьте изменённые файлы в staging area (подготовка к коммиту):

**Добавить конкретный файл:**
```bash
git add Task1SumOfTwoNumbers.java
```

**Добавить все изменённые файлы:**
```bash
git add .
```

**Добавить все файлы из определённой директории:**
```bash
git add seminar-01/tasks/
```

Проверьте статус:

```bash
git status
```

Файлы, готовые к коммиту, будут отображаться зеленым цветом.

---

## Шаг 4: Создание коммита

Создайте коммит с описательным сообщением:

```bash
git commit -m "Решена задача 1: сумма двух чисел"
```

---

## Шаг 5: Push

Отправьте вашу ветку на удалённый репозиторий:

**Первая отправка новой ветки:**
```bash
git push -u origin seminar-01-solutions
```

Флаг `-u` (или `--set-upstream`) связывает локальную ветку с удалённой.

**Последующие отправки:**
```bash
git push
```

После первой отправки с `-u` Git запомнит связь, и можно использовать просто `git push`.
